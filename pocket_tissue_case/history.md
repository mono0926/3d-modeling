# ポケットティッシュケース 設計履歴

## 2026-03-22: フィレット処理のバグ修正と最終適用

### 概要
- `DirectionSelector` に起因するエラー（`'tuple' object has no attribute 'getAngle'`）を解消するため、セレクターを `BoxSelector` に簡略化。
- 各処理を独立した `try-except` で囲むことで、確実に各部（スリット縁、挿入口、内部面取り）へのR加工を適用。
- これにより、手触りと使い勝手を追求した設計意図がすべてモデルに反映された。

## 2026-03-22: 仕上げのブラッシュアップとスリット長の調整

### 概要
- 取り出しやすさを向上させるため、天面スリットの長さを 80mm から **90mm** に変更。
- **仕上げ用フィレットの追加**:
    - 天面スリットの縁に 1.0mm のフィレットを適用し、ティッシュ取り出し時の手触りと滑りを改善。
    - 側面挿入口の縁に 1.0mm のフィレットを適用し、パックの挿入をよりスムーズに。
- 内部のリップ部分の角など、各部のエッジを微調整。

## 2026-03-22: バグ修正（天面全開問題の解消）

### 概要
- 内部くり抜き処理において、カット用のボックスが高すぎて天面を突き抜けていたバグを修正。
- 内部の高さを `INNER_H` (13mm) に厳密に制限することで、天面のリップ形状を正しく保持。

## 2026-03-22: 改良（スリット化と側面保持の強化）

### 概要
- 天面が全開だった問題を解消し、実用的な取り出し用スリット（80mm x 32mm の角丸長方形）に変更。
- 側面の挿入口を左右 4mm ずつ狭め（段差を設け）、ティッシュパックの不意な脱落を防止。

### 設計の工夫
- **Sketch APIの活用**: 角丸のスリットを生成するために `cq.Sketch` を使用し、安定した形状生成を実現。
- **段差の最適化**: 内部寸法 78mm に対して挿入口を 70mm とすることで、適度な保持力を確保。

## 2026-03-22: 初回設計（一体型スリーブ構造）

### 概要
- 内部寸法 110mm x 78mm x 13mm に基づくポケットティッシュケース。
- ユーザーフィラメントにより「蓋なしのシンプルな構造」を採用。
- 上部に 5mm 幅のリップ（返し）を設け、ティッシュの飛び出しを防止。
- 短辺側の一方を開口し、スライド挿入する方式。

### 設計の工夫
- **サポートレス設計**: リップの内側の天井部分に 1mm の面取り（Chamfer）を適用し、45度の傾斜を作ることで 3Dプリンターがサポートなしで印刷しやすくした。
- **堅牢なエッジ処理**: 外周の縦方向のエッジに R2.0 のフィレットを適用し、ポケットなどに入れた際の手触りを考慮。
- **リード角**: 挿入口付近にはわずかな遊びとリード角を意識した設計。

### 使用パラメータ
- `INNER_L`: 110.0
- `INNER_W`: 78.0
- `INNER_H`: 13.0
- `WALL_T`: 2.0
- `LIP_W`: 5.0
- `LIP_T`: 1.5
