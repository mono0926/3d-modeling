# ポケットティッシュケース 設計履歴

## 2026-03-22: 内側フィレットの完全実装と順序最適化
- **課題**: 外周のフィレットをくり抜き前に行っていたため、内部ポケットの角が鋭利なままだった。また、全エッジ一括処理では干渉エラー（BRep_API）が発生していた。
- **対策**:
  1. 外殻作成直後に外周エッジを処理。
  2. 内部くり抜き直後に、Z軸範囲を制限した `BoxSelector` を用いて「内側の垂直4隅」のみを抽出し、確実に R2.0 フィレットを適用。
  3. 内部の底面エッジにも R1.0 フィレットを追加。
- **結果**: すべての仕上げ（外周、内部垂直・底面、スリット、挿入口）がエラーなく適用され、高品質な造形モデルが完成。

## 2026-03-22: 仕上げ処理の徹底修正と堅牢化 (フィレット→面取り)
- **課題**: `BRep_API: command not done` や `'tuple' object has no attribute 'getAngle'` といった幾何学的・セレクター起因のエラーが発生していた。
- **対策**:
  - `fillet(1.0)` をより幾何学的に安定し、3Dプリントにも適した `chamfer(0.5)` (面取り) に変更。
  - `DirectionSelector` が曲線エッジでエラーを起こすため、単純かつ精密な `BoxSelector` に一本化。
  - スクリプト内に自己チェック用のログ出力を追加し、全工程が正常終了することをプログラム的に保証。
- **結果**: すべての仕上げ（内部、天面スリット、挿入口）がエラーなく適用されたことを確認。

## 2026-03-22: 仕上げのブラッシュアップとスリット長の調整
- 取り出しやすさを向上させるため、天面スリットの長さを 80mm から **90mm** に変更。

## 2026-03-22: バグ修正（天面全開問題の解消）
- 内部くり抜き処理において、カット用のボックスが高すぎて天面を突き抜けていたバグを修正。

## 2026-03-22: 改良（スリット化と側面保持の強化）
- 天面が全開だった問題を解消し、実用的な取り出し用スリット（80mm x 32mm の角丸長方形）に変更。
- 側面の挿入口を左右 4mm ずつ狭め（段差を設け）、ティッシュパックの不意な脱落を防止。

## 2026-03-22: 初回設計（一体型スリーブ構造）
- 内部寸法 110mm x 78mm x 13mm に基づくポケットティッシュケース。
- 蓋なしのシンプルな構造を採用。
- **サポートレス設計**: リップの内側の天井部分に面取りを適用し、サポートなしでの印刷を考慮。
